# 实施方案：色卡分类动态管理系统

为了让管理员能够灵活管理色卡仓库的分类，我们将从硬编码的分类列表切换到基于数据库的动态管理系统。

## 1. 数据库架构升级 (Supabase)
*   **创建 `categories` 表**：存储分类名称。
*   **数据迁移**：将现有的硬编码分类（木纹、纯色等）导入数据库作为初始数据。

## 2. 后端服务扩展 (`services/backendService.ts`)
*   **`getCategories()`**：从 Supabase 获取所有分类。
*   **`addCategory(name)`**：新增分类。
*   **`deleteCategory(id)`**：删除分类（需考虑关联数据的处理，这里暂时允许删除，或仅标记删除）。

## 3. 前端界面改造 (`App.tsx`)
*   **状态管理**：引入 `categories` 状态，替换原有的 `CATEGORIES` 常量。
*   **分类管理入口**：在后台侧边栏或顶部增加“分类管理”按钮。
*   **分类管理弹窗/面板**：
    *   列表展示现有分类。
    *   提供“删除”按钮。
    *   提供“新增”输入框。
*   **联动更新**：添加色卡时的“所属系列”下拉框，需改为从 `categories` 状态动态渲染。

这将使你的后台管理功能更加完整和灵活。
